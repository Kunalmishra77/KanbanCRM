Business Logic & Triggers

On stories insert/update/delete:

Insert an activity_log row describing the change (old -> new).

Update stories.updated_at and clients.average_progress using aggregate.

Recompute clients.stage using thresholds:

Hot if average_progress < 30 AND open_stories_count > 2 AND days_since_last_update > 14.

Warm if average_progress < 60 AND days_since_last_update > 7.

Else Cool. (Make thresholds configurable via settings table.)

On comment insert: notify assigned user (push via Realtime event + optional email).

On story moved to Blocked: immediate notification (email + activity log).

Story aging: daily job increments aging metric; surface in dashboard.
